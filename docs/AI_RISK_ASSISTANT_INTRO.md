# AI 风控问答助手介绍

## 1. 功能定位

AI 风控问答助手是 CryptoRisk Agent 系统中的轻量级对话组件，定位为“随时可唤起的金融与加密风险解释助手”。它不是完整的多 Agent 风险报告生成链路，而是嵌入在风控工作台中的即时问答入口，帮助用户在浏览新闻榜单、币种风险榜或分析报告时，快速理解风险分、风险证据、影响范围和后续核验动作。

用户可以通过右下角悬浮按钮打开助手，在聊天窗口中直接提问。助手既能回答通用金融风控问题，也能在问题明确指向当前页面时，结合页面里的新闻、币种、风险分和分析报告进行解释。

## 2. 解决的问题

加密货币风险信息通常具有三个特点：

- 信息来源分散，新闻、公告、链上事件和社区传言混杂在一起。
- 风险判断需要同时理解金融概念、加密资产机制和安全事件语境。
- 用户在查看榜单或报告时，常常需要追问“为什么是这个风险分”“这条证据说明什么”“下一步该核验什么”。

AI 风控问答助手用于补齐这类即时解释能力。它的目标不是替代完整风险分析流程，而是在用户阅读和决策前提供一个专业、克制、上下文感知的解释层。

## 3. 核心能力

### 3.1 金融风险知识问答

助手可以回答利率、汇率、通胀、流动性、杠杆、清算、波动率、信用风险、市场风险、操作风险等基础金融风控问题。

示例问题：

- 稳定币脱锚通常会带来哪些风险？
- 什么是流动性踩踏？
- 杠杆清算为什么会放大市场波动？

### 3.2 加密资产风险解释

助手可以解释 DeFi、跨链桥、L2、交易所、项目基本面、代币经济、链上安全事件、TVL、预言机、合约漏洞等加密资产风险主题。

示例问题：

- 跨链桥被攻击为什么通常是高风险事件？
- 项目方钱包大额转出需要关注哪些信号？
- 预言机异常会影响哪些 DeFi 协议？

### 3.3 当前页面上下文问答

当用户的问题明确指向当前页面、当前新闻、当前币种、风险分、排行榜、证据或报告时，前端会把页面上下文传给后端。助手会基于这些上下文解释页面结果。

可使用的上下文包括：

- 当前视图类型，例如新闻榜、币种榜或聊天分析结果。
- 当前主题资产或新闻。
- 当前风险分和风险等级。
- 最新生成的风险报告。
- 当前榜单中的头部新闻和头部币种。

示例问题：

- 当前页面里这个币为什么风险分这么高？
- 这条新闻里的主要证据是什么？
- 这个报告还缺哪些关键信息？

### 3.4 风控核验建议

当信息不足或事件尚未确认时，助手会给出核验清单，而不是直接下结论。

常见核验方向包括：

- 是否有官方公告或交易所公告。
- 是否存在链上交易哈希、攻击地址、资金流向。
- 是否有安全团队复盘或审计机构说明。
- 风险是否已经影响提现、交易、抵押品、稳定币锚定或协议流动性。

## 4. 产品交互设计

助手采用右侧抽屉式聊天窗口：

- 入口：页面右下角悬浮按钮。
- 标题：AI风控助手。
- 副标题：金融与加密风险问答。
- 输出方式：流式生成，降低等待感。
- 内容展示：支持基础 Markdown，包括标题、加粗、列表和行内代码。
- 失败提示：当服务不可用时，返回“助手暂时无法回答，请稍后重试”。

这种设计适合风控工作台场景：用户不需要离开当前榜单或报告页面，就可以继续追问、解释和核验。

## 5. 技术实现

### 5.1 前端链路

前端组件位于：

```text
frontend/components/risk-dashboard.tsx
```

核心逻辑包括：

- 维护助手消息列表、输入框状态、加载状态和错误状态。
- 根据用户问题判断是否需要附带页面上下文。
- 调用流式接口 `/api/risk-assistant/stream`。
- 将后端返回的文本分片实时追加到助手消息中。

前端并不会无条件把页面数据塞给模型。它会先通过问题内容判断用户是否真的在问当前页面，避免“用户问通用金融问题，但模型被页面里的币种或新闻带偏”。

### 5.2 上下文选择策略

助手的上下文注入采用“按需附带”策略。

当问题包含以下类型表达时，会附带页面上下文：

- 当前页面、这个页面、这里、这个、这条、这篇、上面等指代词。
- 风险分、风险评分、排行榜、榜单、证据、报告等页面数据词。
- 问题中明确出现当前币种或新闻相关资产，并且询问风险分、证据或报告。

如果用户只是问一般金融或加密知识问题，前端会传入一个轻量上下文，说明页面上下文可用但本次已省略。这样可以减少上下文污染，让模型优先回答用户本身的问题。

### 5.3 后端链路

后端服务位于：

```text
backend/app/services/risk_assistant_service.py
```

主要函数包括：

- `build_risk_assistant_prompt`：组装用户问题和页面上下文。
- `answer_risk_assistant`：普通非流式问答。
- `stream_risk_assistant_answer`：流式问答。

FastAPI 暴露两个接口：

```text
POST /api/risk-assistant
POST /api/risk-assistant/stream
```

其中流式接口使用 `text/event-stream` 返回内容分片，前端可以边接收边展示。

### 5.4 模型调用

助手通过 DeepSeek API 调用 `deepseek-v4-pro`。系统提示词定义了助手的角色、可回答范围、回答边界和输出风格。

模型侧重点包括：

- 使用中文回答。
- 先给结论，再给 2 到 4 个要点。
- 优先做风险解释、知识科普、情景分析和核验清单。
- 不输出买入、卖出、做多、做空、价格点位或收益承诺。
- 不编造实时价格、实时新闻或上下文没有提供的事实。
- 信息不足时明确说明“不足以判断”。

## 6. 风控边界

AI 风控问答助手有明确的安全边界：

- 不提供投资建议。
- 不给出交易方向。
- 不承诺收益。
- 不编造实时行情和实时新闻。
- 不把未经确认的信息说成事实。
- 不因为上下文中出现某个币种，就强行把用户问题改写为该币种问题。
- 对高风险问题优先提示风险来源和不确定性。

这些约束让助手更像一个风控解释员，而不是交易顾问。

## 7. 与主 Multi-Agent 链路的区别

AI 风控问答助手和主风险分析 Agent 的职责不同：

- 主 Multi-Agent 链路负责对输入事件生成结构化风险报告，包括风险识别、分类、证据抽取、评分、影响分析和处置建议。
- AI 风控问答助手负责对用户追问进行即时解释，包括金融知识问答、页面结果解释和核验建议。

二者互补：

- 主链路产出结构化结果。
- 小助手帮助用户理解结果、追问细节、形成下一步核验思路。

## 8. 典型使用场景

### 场景一：解释风险分

用户在新闻风险榜中看到某条新闻风险分较高，可以问：

```text
这条新闻为什么是高风险？
```

助手会结合当前新闻、风险分、证据和摘要，解释主要风险信号。

### 场景二：理解加密风险概念

用户看到“跨链桥攻击”但不理解影响，可以问：

```text
跨链桥攻击为什么容易造成系统性风险？
```

助手会从资产托管、跨链流动性、锚定资产、资金外流和信任传导等角度解释。

### 场景三：生成核验清单

当新闻还没有官方确认时，用户可以问：

```text
这件事现在还需要核验哪些信息？
```

助手会列出官方公告、链上交易、攻击地址、资金流向、安全团队复盘等核验点。

## 9. 项目亮点

- 嵌入式问答：不离开风控工作台即可追问。
- 流式输出：提升交互体验，适合聊天窗口。
- 上下文感知：能解释当前页面中的风险分、证据和榜单结果。
- 上下文克制：只有问题明确相关时才引用页面数据，减少答非所问。
- 风控边界清晰：只做解释、科普、核验和风险提示，不做投资建议。
- 降级提示友好：模型不可用时返回清晰错误信息，避免前端空白。

## 10. 后续优化方向

- 增加多轮上下文记忆，支持连续追问同一条新闻或同一个币种。
- 接入链上浏览器、安全公司报告和交易所公告，增强事实核验能力。
- 为助手增加引用来源展示，让回答中的证据更可追溯。
- 增加常见风控问题模板，例如“解释风险分”“列出核验点”“总结影响范围”。
- 增加回答质量评估，对投资建议、事实编造和上下文误用进行自动检测。
